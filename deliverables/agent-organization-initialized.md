## 🔄 標準的な開発フロー

```
1. Project Manager → タスク計画
           ↓
2. Architect/Design → 設計
           ↓
3. Frontend/Backend/AI → 並行実装
           ↓
4. Code Reviewer → レビュー
           ↓
5. Test → テスト実行
           ↓
6. DevOps → デプロイ
           ↓
7. Documentation → ドキュメント更新
           ↓
8. Project Manager → 進捗記録
```

## 💡 重要な注意事項

### エージェント運用のベストプラクティス

1. **明確なタスク定義**
   - 各エージェントには具体的で測定可能なタスクを割り当てる
   - 期待する成果物を明確に指定

2. **適切なコミュニケーション**
   - エージェント間の依存関係を明確にする
   - ブロッカーは即座にProject Managerに報告

3. **進捗の可視化**
   - 週次レポートで全体進捗を把握
   - エージェント活動ログを記録

4. **品質の維持**
   - Code Reviewerを必ず通す
   - テストカバレッジ80%以上を維持

5. **ドキュメントの更新**
   - 主要な変更はDocumentation Agentに報告
   - 技術的決定はmemoryに記録
